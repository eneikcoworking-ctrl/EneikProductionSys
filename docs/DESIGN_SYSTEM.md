# Design System (Eneik Visual Standard)

## Введение
Данный документ определяет стандарты визуальной эстетики и технической реализации интерфейсов в системе Eneik. В то время как `Role Excellence Charter` фокусируется на юзабилити и когнитивной эргономике, Design System гарантирует эстетическую цельность и профессиональный вид (B2B SaaS look & feel).

Документ является расширением и дополнением к ролям **BARCAN-TAG-03** (Design) и **BARCAN-TAG-11** (Frontend).

## Типографическая шкала
Используется модульная сетка с коэффициентом **1.25 (Major Third)**. Базовый размер (body) = `16px` (`1rem`). Шрифты должны быть масштабируемыми.

| Уровень | Размер (rem) | Размер (px) | Вес (Weight) |
|---------|--------------|-------------|--------------|
| **h1** | `3.815rem` | ~61px | 800 |
| **h2** | `3.052rem` | ~49px | 800 |
| **h3** | `2.441rem` | ~39px | 700 |
| **h4** | `1.953rem` | ~31px | 700 |
| **h5** | `1.563rem` | ~25px | 600 |
| **h6** | `1.25rem` | 20px | 600 |
| **body** | `1rem` | 16px | 400 |
| **caption**| `0.8rem` | ~13px | 400 |

## Система отступов (Spacing)
В основе лежит **8pt Grid**. Все отступы (padding, margin, gap) должны быть кратны базовой единице `4px` (0.25rem) или `8px` (0.5rem).

| Token | rem | px | Применение |
|-------|-----|----|------------|
| **space-1** | `0.25rem` | 4px | Минимальные зазоры |
| **space-2** | `0.5rem` | 8px | Внутренние отступы малых элементов |
| **space-3** | `0.75rem` | 12px | Padding в карточках |
| **space-4** | `1rem` | 16px | Стандартный gap в сетках |
| **space-6** | `1.5rem` | 24px | Внешние отступы блоков |
| **space-8** | `2rem` | 32px | Разделение секций |

## Библиотека компонентов — источник истины
Все переиспользуемые UI-компоненты (Buttons, Cards, Inputs, Modals) физически располагаются в директории:
`frontend/src/components/`

**Правило создания:** Если нужного компонента нет в библиотеке, он создаётся там как атомарный элемент. Прямое дублирование стилей в разных представлениях (Views) запрещено.

## Deontic-классификация
Данная классификация согласуется с форматом Role Excellence Charter и обязательна для исполнения.

- **Obligatory**: Использование цветовых токенов, типографической шкалы и системы отступов, определенных выше, для любого нового UI. Компоненты должны наследоваться от существующих в `frontend/src/components/`.
- **Forbidden**:
    - Использование инлайновых стилей (`style="..."`) с произвольными значениями цвета или отступов.
    - Использование единиц измерения `px` для размеров шрифтов (только `rem`).
    - Отклонение от 8pt сетки (например, `margin: 15px`).
- **Permitted**:
    - Использование "brand exception" для уникальных маркетинговых элементов или специальных AI-визуализаций.
    - В этом случае обязателен технический комментарий в коде: `/* BRAND EXCEPTION: [Обоснование необходимости отклонения] */`.

## Design Excellence Gate Integration
Текущая реализация `src/main/java/com/eneik/production/services/gate/DesignExcellenceGate.java` проверяет базовую доступность и адаптивность (`responsive_ok`, `visual_qa_ok`).

Настоящий документ является спецификацией для будущего критерия автоматической проверки:
**`design_token_compliance`** — проверка на использование только разрешенных CSS переменных или HEX-кодов из палитры. До внедрения автоматики, соблюдение Design System проверяется в рамках Code Review на соответствие данному стандарту.
