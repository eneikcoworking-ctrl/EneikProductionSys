# AGENT BACKLOG: 7 CRITICAL PRODUCTION TASKS (OS MVP)

These tasks are designed for AI Agents to execute independently within their respective domain boundaries. All tasks target the "Production OS" MVP to enable the Agentic Handshake.

---

## TASK 1: CORE DOMAIN ENTITIES & PERSISTENCE (JAVA)
- **Agent**: `BARCAN-TAG-01` (Architect) & `BARCAN-TAG-08` (Data Engineer)
- **Domain**: `src/main/java/com/eneik/production/models/` & `src/main/resources/db/migration/`
- **Description**: Implement JPA entities for `Account`, `AgentRole`, `Task`, and `Assignment`. Create Flyway migrations for the corresponding database tables.
- **Isolation Strategy**: Only touches the `models` and `db/migration` directories. No controller or view changes.
- **DOD**: `mvn test` passes with H2-backed integration tests for the new entities.

## TASK 2: REQUIREMENT DECOMPOSITION SERVICE (JAVA)
- **Agent**: `BARCAN-TAG-09` (Mediator)
- **Domain**: `src/main/java/com/eneik/production/services/`
- **Description**: Port the rule-based decomposition logic from the TypeScript prototype to a Java service. It should take a requirement string and return a list of `Task` entities with suggested `BARCAN-TAG-XX`.
- **Isolation Strategy**: Strictly within the `services` layer. Depends on models from Task 1.
- **DOD**: Unit test confirms correct tag assignment for "UI", "Database", and "Security" keywords.

## TASK 3: TASK LIFECYCLE REST CONTROLLER
- **Agent**: `BARCAN-TAG-02` (Backend Engineer)
- **Domain**: `src/main/java/com/eneik/production/controllers/tasks/`
- **Description**: Implement REST endpoints: `GET /api/v1/tasks/available` (unassigned tasks), `POST /api/v1/tasks/{id}/claim` (assign to current account), and `PATCH /api/v1/tasks/{id}/status`.
- **Isolation Strategy**: Limited to the `controllers/tasks` package. Uses services from Task 2.
- **DOD**: Swagger/OpenAPI documentation reflects all three endpoints with correct status codes.

## TASK 4: PRODUCTION OS DASHBOARD (SVELTE)
- **Agent**: `BARCAN-TAG-11` (Frontend) & `BARCAN-TAG-03` (Designer)
- **Domain**: `frontend/src/components/dashboard/`
- **Description**: Build a Svelte dashboard visualizing the production pipeline. Show active tasks, agent workload, and a "Claim Task" button for the current user/agent.
- **Isolation Strategy**: Works only within the `frontend` directory. Communicates with Backend via established API contracts.
- **DOD**: Dashboard renders in dev mode and correctly displays a list of tasks fetched from the API.

## TASK 5: LINEAR SYNCHRONIZATION WRITE-BACK
- **Agent**: `BARCAN-TAG-05` (DevOps)
- **Domain**: `scripts/linear_sync.py`
- **Description**: Update the sync script to support two-way synchronization. It must be able to create new Linear issues from the local `Task` entities generated by the Decomposition Service.
- **Isolation Strategy**: Only modifies Python scripts in the `scripts/` folder.
- **DOD**: Running the script successfully creates a task in a test Linear project with the correct BARCAN label.

## TASK 6: BOTTLENECK & CAPACITY ANALYTICS
- **Agent**: `BARCAN-TAG-04` (ML/Data Scientist)
- **Domain**: `src/models/ml/PredictionService.py`
- **Description**: Implement an algorithm to detect system bottlenecks. Analyze the task queue to identify which agent roles are over-capacity or causing delays in the TOC pipeline.
- **Isolation Strategy**: Limited to the FastAPI ML service.
- **DOD**: The `/api/v1/predict/bottleneck` endpoint returns a JSON report identifying the current constraint (e.g., "TAG-00 is the bottleneck").

## TASK 7: AGENTIC BOUNDARY GUARD (JULES)
- **Agent**: `BARCAN-TAG-00` (Code Guardian)
- **Domain**: `scripts/audit_pr.py`
- **Description**: Implement the "Boundary Enforcement" logic in the audit script. It must check the `agentTag` of the task being worked on and fail the PR if files outside that agent's directory map (defined in `DIRECTORY_MAP.md`) are modified.
- **Isolation Strategy**: Only modifies the audit script.
- **DOD**: Script correctly flags a PR as "Violation" if a TAG-11 (Frontend) agent tries to modify a Java Controller.
