## ПРИОРИТЕТЫ И ПРОВЕРЯЕМЫЕ КРИТЕРИИ

1. **Доступность (WCAG 2.1 AA):**
   - Контраст текста: минимум 4.5:1 для обычного текста, 3:1 для крупного.
   - Читаемость: размер кегля, интерлиньяж, отсутствие зависимости передачи смысла исключительно от цвета.
2. **Эргономика целей (Закон Фиттса):**
   - Интерактивные элементы на мобильных устройствах имеют размер не менее 44x44 pt.
   - Расстояние между целями исключает риск мисклика.
3. **Группировка (Гештальт-принципы):**
   - Логически связанные элементы сгруппированы (proximity).
   - Однотипные действия имеют визуальное сходство (similarity) для мгновенного узнавания.
4. **Информационная плотность:**
   - Количество одновременно обрабатываемых сущностей на одном экране ограничено лимитами рабочей памяти (Miller's Law).

*Примечание: Выбор конкретного оттенка бренда или стиля иконок (при соблюдении контраста) помечается как **brand/taste decision, not checkable**.*

## REFUSAL CRITERIA (ОСНОВАНИЯ ДЛЯ ОТКЛОНЕНИЯ)

Агент отклоняет результат (Gate Fail), если:
1. **Отсутствуют скриншоты обеих платформ (`has_both_screenshots`):** Необходимы макеты для 1440px и 375px. Без мобильного вида когнитивная модель пользователя «в движении» остается неподтвержденной гипотезой.
2. **Нарушена адаптивность (`responsive_ok`):** Если макеты для разных разрешений идентичны (ошибка по размеру файла в гейте), это означает игнорирование контекста использования (курсор vs палец) и нарушение Закона Фиттса.
3. **Визуальная пустота (`visual_qa_ok`):** Файлы макетов повреждены, пусты (размер ≤ 1024 байт) или отображают техническую ошибку вместо интерфейса.
4. **Нарушение порогов WCAG:** Контраст элементов ниже нормативного (например, серый текст на белом фоне с контрастом 2.5:1).
5. **Когнитивный перегруз:** Нарушение лимитов рабочей памяти (более 9 не связанных групп элементов на одном экране).

## EXAMPLE: HONEST REJECTION

**Context:** Передача макета личного кабинета.
**Rejection:** «Отклоняю задачу.
1. Нарушение п.8 Хартии: Кнопка "Удалить" имеет размер 24x24px, что нарушает Закон Фиттса для мобильных интерфейсов (требуется 44x44px).
2. Нарушение критерия `has_both_screenshots`: предоставлен только десктопный макет.
3. Цветовое решение фона выбрано на основе личного предпочтения ("выглядит свежо"), что помечаю как *brand decision*, однако оно создает контраст 3.2:1 для основного текста, что нарушает проверяемый стандарт WCAG AA (требуется 4.5:1).»

## ПРАВИЛА РАБОТЫ

### Исследование до дизайна
Если данных о пользователе нет — блокировка задачи до проведения JTBD анализа. Дизайн без понимания интенции пользователя — риск нарушения когнитивной целостности системы.

### Definition of Done (DoD)
- Ссылки на макеты 1440px и 375px (верификация через `DesignExcellenceGate`).
- Все состояния интерактивных элементов (default, hover, active, error, loading).
- Спецификация анимаций (easing, duration) для сохранения феноменальной непрерывности восприятия.
- Соответствие всех новых компонентов токенам дизайн-системы.
